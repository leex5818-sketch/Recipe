## 1. 쿠팡 파트너스 — 첫 수익 점등

### 1-1. 가입·승인
1. **가입 URL**: https://partners.coupang.com → 쿠팡 계정으로 로그인 → 파트너스 신청
2. **채널(웹사이트) 등록**: 대시보드 → "채널 관리" → `https://recipe-lac-ten.vercel.app` 등록
3. **승인 대기**: 보통 며칠. 승인 전엔 링크가 동작하지 않으니 트래픽 약간 만든 뒤 신청 권장
4. **요금**: 가입 무료. 식품 카테고리 수수료 **약 3%**(카테고리별 상이)

### 1-2. ⚠️ AF_ID 반드시 본인 것으로 교체
현재 코드(`src/app.jsx`)에 `COUPANG_AF_ID="AF7478113"` 가 박혀 있습니다.
**이게 본인 파트너스 ID가 아니면 수수료가 남에게 갑니다.**
- 파트너스 대시보드에서 본인 트래킹 ID 확인 → `src/app.jsx` 의 `COUPANG_AF_ID` 교체 → `npm run build` → 커밋.

### 1-3. 링크 동작 방식 (중요)
현재 앱은 재료 클릭 시 **쿠팡 검색 URL + 제휴 파라미터**로 보냅니다(폴백). 하지만 **정확한 수수료 적립은 파트너스가 발급한 공식 딥링크(`link.coupang.com/...`)로 해야 신뢰**할 수 있습니다.
- 파트너스 대시보드 "링크 생성"에서 상품/검색 딥링크를 만들어 `coupang_links.json` 에 채우면, 앱이 검색 URL 대신 그 딥링크를 우선 사용합니다.
- **우선순위 재료**(가장 많이 쓰임 → 딥링크 ROI 최대): `scripts/top_ingredients.py` 로 뽑은 상위 30개가 `coupang_links.template.json` 에 들어 있음. 예: 대파(53)·소금(51)·양파(41)·식용유(36)·간장(32)·마늘(30)…
- 채우는 법:
  ```
  python3 scripts/top_ingredients.py --top 50 --write-template   # 템플릿 생성
  # coupang_links.template.json 의 각 재료 값에 파트너스 딥링크 붙여넣기
  mv coupang_links.template.json coupang_links.json              # 앱이 자동 로드
  npm run build && (커밋·배포)
  ```
- (자동화) 파트너스 **Open API** 키가 있으면 딥링크 일괄 생성 스크립트로 30~50개를 한 번에 만들 수 있음 — 필요 시 요청.

### 1-4. 현실 수익 추정
수익 = **트래픽 함수**.
- 일 100명 × 재료클릭 20% × 구매전환 2% × 객단가 1.5만 × 수수료 3% ≈ **월 5천 원대**
- 일 1,000명까지 키워야 **월 5만 원대**
- → 쿠팡만으로 큰돈은 어려움. **트래픽을 키우는 것 + 아래 프리미엄**이 핵심.

## 2. 평생 프리미엄 ₩9,900 — 진짜 수익 (설계)

한국은 구독 거부감이 커서 **일회성 "평생 프리미엄"**이 유리(baby-weekly-guide 선례). 광고 없이, 한 번 결제로 평생.

### 2-1. 유료화할 기능 (무료 vs 프리미엄)
| 기능 | 무료 | 프리미엄 |
|------|------|----------|
| 레시피 열람·번역·즐겨찾기 | ✅ | ✅ |
| **주간 식단 플래너** (요일별 메뉴 짜기) | 미리보기 | ✅ 전체 |
| **통합 장보기 리스트** (여러 레시피 재료 합산 → 쿠팡 일괄) | 1개 레시피 | ✅ 무제한 합산 |
| **광고 제거** | 광고 노출 | ✅ 제거 |
| 영양정보·n인분 자동 환산(추후) | — | ✅ |

> 장보기·식단 기능은 **쿠팡 전환율도 같이 끌어올리는 이중 구조** — 프리미엄 수익 + 제휴 수익 동시.

### 2-2. 결제 연동 (토스페이먼츠 — 국내 표준 스택)
- **토스페이먼츠** 가맹점 가입(https://www.tosspayments.com) → 테스트 키 발급
- 정적 PWA라 **결제 승인은 서버가 필요** → Supabase **Edge Function**으로 결제 승인 + 권한 부여:
  1. 클라이언트: 토스 결제위젯 → 결제 요청 → `paymentKey`/`orderId`/`amount` 수신
  2. Edge Function: 토스 `/v1/payments/confirm` 호출(시크릿 키) → 성공 시 `users` 테이블에 `premium=true` 기록(현재 카카오 `kakao_id` 기준)
  3. 앱: 로그인 사용자의 `premium` 플래그로 기능 잠금 해제
- **소유자 가드**: 키는 env, 권한은 서버(Edge Function)에서만 부여 — 기존 owner-key 패턴과 동일 원칙.

### 2-3. 구현 로드맵 (코드 작업)
1. (선행) docs/SECURITY.md 의 **Edge Function** 골격 재사용 — 결제 승인 함수 추가
2. `users` 스키마에 `premium boolean default false` 컬럼 + 결제 승인 시 service_role로 set
3. 앱: `user.premium` 상태 → 식단 플래너/장보기 잠금 해제 + 페이월 UI
4. 장보기 리스트 기능 구현(여러 레시피 `ing` 합산 → 재료별 쿠팡 딥링크 + "전체 담기")
5. 토스 결제위젯 연동 + Edge Function confirm

> 1~5는 별도 작업 단위. 원하면 "프리미엄 만들어줘"로 착수 — Edge Function·페이월·장보기까지 한 번에.
